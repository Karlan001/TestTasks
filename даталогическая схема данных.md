# Даталогическая схема данных

#### 1. Таблица clients (клиенты)

- id (PRIMARY KEY) — уникальный идентификатор клиента (тип: INTEGER);
- name (VARCHAR) — имя клиента (тип: строка, A-Z);
- address (VARCHAR) — адрес клиента (тип: строка, A-Z).

*Назначение: хранение информации о клиентах, которые делают заказы.*

#### 2. Таблица orders (заказы)

- id (PRIMARY KEY) — уникальный идентификатор заказа (тип: INTEGER);
- number (INTEGER) — номер заказа;
- client_id (FOREIGN KEY) — ID клиента, сделавшего заказ (ссылка на clients.id);
- numenclature_id (FOREIGN KEY) — ID номенклатуры (товара), включённого в заказ (ссылка на numenclature.id).

*Назначение: хранение информации о заказах, включая связь с клиентом и номенклатурой.*

#### 3. Таблица numenclature (номенклатура — товары)

- id (PRIMARY KEY) — уникальный идентификатор номенклатуры (товара) (тип: INTEGER);
- title (VARCHAR) — название товара (тип: строка, A-Z);
- quantity (INTEGER) — количество товара (тип: число, 123);
- price (DECIMAL/FLOAT) — цена товара (тип: число, 123);
- numenclature_catalog_id (FOREIGN KEY) — ID каталога номенклатуры (ссылка на numenclature_catalog.id).

*Назначение: хранение информации о товарах (номенклатуре), включая название, количество, цену и принадлежность к каталогу.*

#### 4. Таблица numenclature_catalog (каталог номенклатуры)

- id (PRIMARY KEY) — уникальный идентификатор каталога номенклатуры (тип: INTEGER);
- title (VARCHAR) — название каталога (тип: строка, A-Z);
- perant_category (VARCHAR) — родительская категория (тип: строка).

*Назначение: хранение категорий (каталогов) для группировки номенклатуры (товаров).*

## Связи между таблицами
#### **Связь clients ↔ orders:**

**Тип:** *один ко многим (1:N).*

**Описание:** *один клиент (clients) может иметь множество заказов (orders), каждый заказ связан с одним клиентом через client_id.*

#### **Связь numenclature ↔ orders:**

**Тип:** *один ко многим (1:N).*

**Описание:** *один товар (numenclature) может быть включён в множество заказов (orders), каждый заказ содержит один товар через numenclature_id.*

#### **Связь numenclature ↔ numenclature_catalog:**

**Тип:** *многие к одному (N:1).*

**Описание:** *множество товаров (numenclature) относятся к одному каталогу (numenclature_catalog), связь через numenclature_catalog_id.*

## Итог
#### **Даталогическая схема описывает структуру базы данных с четырьмя таблицами и связями между ними:**
- клиенты делают заказы (clients → orders);
- заказы содержат номенклатуру (товары) (orders → numenclature);
- номенклатура (товары) относятся к определённым каталогам (numenclature → numenclature_catalog).
- Такая схема позволяет эффективно хранить и обрабатывать данные о клиентах, их заказах, товарах и их категоризации.
